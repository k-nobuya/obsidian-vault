# セカンドブレイン構築メモ

作成日: 2026-06-16

---

## 全体像

AIとの会話を自動でObsidianに保存する仕組みを作っている。

```
Claude（AI）
    ↓ 会話内容をWebhookで送信
Make.com（自動化ツール）
    ↓ GitHubのAPIでファイルを作成
GitHub（クラウド保管庫）
    ↓ Gitプラグインで同期
Obsidian（ローカルのノートアプリ）
```

---

## 各ツールの役割

| ツール | 役割 |
|--------|------|
| Claude | 思考の整理・対話相手 |
| Make.com | 自動化の司令塔。Webhookでデータを受け取りGitHubに保存 |
| GitHub | クラウド上のファイル置き場。PrivateリポジトリでVaultを管理 |
| Obsidian | ローカルのノートアプリ。GitHubと同期して最新メモを受け取る |

---

## やったこと

### ① Make.comに登録
- 無料アカウントを作成
- 新しいScenarioを作成（ai-memoという名前のWebhookを設定）

### ② ObsidianとGitHubを繋ぐ
- Obsidian GitプラグインをインストールしてON
- GitHubに`obsidian-vault`というPrivateリポジトリを作成
- Mac上の`~/Documents/Obsidian Vault`フォルダをGitと連携
- GitHubにSSH鍵を登録（セキュアな接続のため）
- `git push`でVaultの内容をGitHubにアップロード

### ③ Make.comとGitHubを繋ぐ
- Make.comにWebhookモジュールを追加（外部からデータを受け取る口）
- HTTPモジュールでGitHub APIを叩いてファイルを作成する設定
- GitHub Personal Access Tokenで認証

---

## メモを保存する現在の手順

1. メモしたい内容をBase64に変換する

```bash
echo -n "メモの内容" | base64
```

2. Make.comのWebhookにPOSTで送信

```bash
curl -X POST https://hook.eu1.make.com/uuvj4x88lb6157hi77m7gx6dlb7nktye \
  -H "Content-Type: application/json" \
  -d '{"content": "Base64に変換した文字列"}'
```

3. GitHubのobsidian-vaultリポジトリにmdファイルが追加される
